📊 Báo cáo Kiểm thử Hệ thống Quản lý Kho hàng

Ngày tạo: 19/3/2026

Người thực hiện: Nguyễn Thanh Tùng

1. Tổng quan

Hệ thống quản lý kho hàng (Inventory Management System) cung cấp các API endpoint cho việc quản lý sản phẩm và kho hàng.

Công cụ kiểm thử: Postman

Máy chủ: localhost:3000

2. Cấu trúc Database

Product Collection

{
  "_id": ObjectId,
  "name": string,
  "description": string,
  "price": number,
  "createdAt": Date
}

Inventory Collection

{
  "_id": ObjectId,
  "product": ObjectId (ref: Product),
  "stock": number (≥ 0),
  "reserved": number (≥ 0),
  "soldCount": number (≥ 0),
  "createdAt": Date
}

3. Các API Endpoint

A. Product Management

POST /api/products

Mô tả: Tạo sản phẩm mới. Khi tạo sản phẩm, hệ thống sẽ tự động tạo inventory tương ứng.

Request Body:

{
  "name": "Laptop Dell",
  "description": "High-performance laptop",
  "price": 1200
}

Response:

{
  "product": { ... },
  "inventory": { ... }
}
GET /api/products

Mô tả: Lấy danh sách tất cả sản phẩm

Response: Mảng các sản phẩm

GET /api/products/:id

Mô tả: Lấy chi tiết sản phẩm theo ID

Parameters: id - Product ID

B. Inventory Management

GET /api/inventory

Mô tả: Lấy danh sách tất cả kho hàng (với join thông tin sản phẩm)

Response: Mảng inventory kèm product details

GET /api/inventory/:id

Mô tả: Lấy chi tiết kho hàng theo ID (với join sản phẩm)

Parameters: id - Inventory ID

GET /api/inventory/product/:productId

Mô tả: Lấy kho hàng theo Product ID

Parameters: productId - Product ID

POST /api/inventory/add-stock

Mô tả: Thêm stock cho sản phẩm

Request Body:

{
  "product": "product_id",
  "quantity": 100
}

Hiệu ứng: stock += quantity

POST /api/inventory/remove-stock

Mô tả: Giảm stock cho sản phẩm (kiểm tra đủ stock)

Request Body:

{
  "product": "product_id",
  "quantity": 10
}

Hiệu ứng: stock -= quantity (nếu stock ≥ quantity)

⚠️ Lỗi nếu stock không đủ

POST /api/inventory/reservation

Mô tả: Tạo đặt hàng - giảm stock, tăng reserved

Request Body:

{
  "product": "product_id",
  "quantity": 20
}

Hiệu ứng: stock -= quantity, reserved += quantity

⚠️ Kiểm tra đủ stock

POST /api/inventory/sold

Mô tả: Ghi nhận hàng đã bán - giảm reserved, tăng soldCount

Request Body:

{
  "product": "product_id",
  "quantity": 5
}

Hiệu ứng: reserved -= quantity, soldCount += quantity

⚠️ Kiểm tra đủ reserved

4. Kế hoạch Kiểm thử

✓ Test Case 1: Tạo Sản phẩm

  1. Gọi endpoint: POST /api/products
  2. Gửi dữ liệu: 
    { "name": "Laptop Dell", "description": "High-performance", "price": 1200 }
  3. Xác nhận: Product được tạo và inventory tự động được tạo
  4. Mã lưu: product_id cho các test tiếp theo

✓ Test Case 2: Lấy Danh sách Inventory

  1. Gọi endpoint: GET /api/inventory
  2. Xác nhận: Trả về danh sách inventory kèm thông tin sản phẩm
  3. Lưu inventory_id cho các test tiếp theo

✓ Test Case 3: Thêm Stock

  1. Gọi endpoint: POST /api/inventory/add-stock
  2. Gửi: { "product": "product_id", "quantity": 100 }
  3. Xác nhận: stock tăng từ 0 lên 100

✓ Test Case 4: Giảm Stock

  1. Gọi endpoint: POST /api/inventory/remove-stock
  2. Gửi: { "product": "product_id", "quantity": 20 }
  3. Xác nhận: stock giảm từ 100 xuống 80

✓ Test Case 5: Tạo Reservation

  1. Gọi endpoint: POST /api/inventory/reservation
  2. Gửi: { "product": "product_id", "quantity": 30 }
  3. Xác nhận: stock giảm, reserved tăng (stock: 50, reserved: 30)

✓ Test Case 6: Record Sold

  1. Gọi endpoint: POST /api/inventory/sold
  2. Gửi: { "product": "product_id", "quantity": 15 }
  3. Xác nhận: reserved giảm, soldCount tăng (reserved: 15, soldCount: 15)

✓ Test Case 7: Lấy Inventory theo ID

  1. Gọi endpoint: GET /api/inventory/inventory_id
  2. Xác nhận: Trả về thông tin chi tiết kèm product (join)

5. Hướng dẫn Kiểm thử với Postman

Bước 1: Import Collection

  1. Mở Postman
  2. Nhấn "Import" → "Choose Files"
  3. Chọn file Inventory_API.postman_collection.json
  4. Bộ sưu tập sẽ được import

Bước 2: Đặt Biến Environment

  1. Tạo Environment mới với tên "Inventory"
  2. Thêm biến:
    Biến Giá trị
    product_id (sẽ cập nhật sau khi tạo product)
    inventory_id (sẽ cập nhật sau khi tạo inventory)

Bước 3: Chạy các Test

  1. Khởi động server: npm start
  2. Tạo sản phẩm: POST /api/products
    • Sao chép product._id vào biến product_id
  3. Lấy danh sách inventory: GET /api/inventory
    • Sao chép inventory._id vào biến inventory_id
  4. Chạy các endpoint còn lại: Thêm stock, giảm stock, reservation, sold
  5. Kiểm tra kết quả: Xác nhận các giá trị thay đổi đúng theo yêu cầu

6. Kết quả Kiểm thử

Test Case Kết quả Ghi chú
Tạo sản phẩm ✓ PASS Product và Inventory được tạo tự động
Lấy danh sách ✓ PASS Trả về dữ liệu kèm join thông tin
Thêm stock ✓ PASS Stock tăng chính xác
Giảm stock ✓ PASS Stock giảm chính xác, kiểm tra đủ
Reservation ✓ PASS Stock giảm, reserved tăng
Sold ✓ PASS Reserved giảm, soldCount tăng
Join Product ✓ PASS Thông tin sản phẩm được trả về
